# 剩余优化与功能拓展清单

本文档用于记录当前项目后续可继续推进的优化项和功能拓展。已完成的拆分架构、渲染安全优化、未使用代码清理、模型请求高级参数、多模型配置持久化与前端切换不再列入本清单。

## 工程化优化

### 配置与环境管理

- 增加 `.env` 文件加载能力，减少本地开发时反复导出环境变量的成本。
- 新增 `.env.example`，明确 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_CHAT_MODEL`、`OPENAI_MAX_TOKENS` 等配置项。
- 将不同模型服务商的默认配置整理为预设，例如 DeepSeek、OpenAI、通义千问兼容接口等。

### 测试与代码质量

- 引入 `pytest`，覆盖消息清洗、上下文裁剪、会话管理、配置解析、错误提示映射等核心逻辑。
- 引入 `ruff` 或 `black`，统一格式化和基础静态检查。
- 在 README 中补充开发命令，例如测试、格式化、启动应用。

### 日志与可观测性

- 增加基础日志，记录模型请求耗时、是否命中重试、错误类型等信息。
- 在页面展示本轮请求耗时、模型名称、上下文消息数量等轻量状态。
- 后续可扩展 token 用量统计和费用估算。

## 数据与会话能力

### 聊天记录持久化

- 使用 PostgreSQL 保存会话和消息，避免刷新页面或重启服务后丢失记录。
- 支持会话重命名、删除确认、按时间排序。
- 支持导出当前会话为 Markdown 或 JSON。

### 多用户支持

- 如果项目从本地 Demo 演进为团队工具，需要增加用户身份、会话归属和访问隔离。
- 可先使用简单用户标识，后续再接入正式登录体系。

## 模型与能力拓展

### 多服务商与多模型管理

- 增加模型服务商预设，减少手动输入 Base URL 和模型名。
- 支持同一问题调用多个模型，对比回答质量、耗时和错误情况。

### Prompt 模板库

- 提供常用系统提示词模板，例如写作助手、代码审查、SQL 助手、客服助手、学习教练。
- 支持保存、编辑、复制和应用模板。
- 模板可包含变量占位符，后续用于更稳定的任务型对话。

### RAG 知识库问答

- 支持上传 Markdown、TXT、PDF 等文件。
- 增加文档切分、向量化、检索和引用来源展示。
- 先实现本地文件知识库，后续再考虑接入数据库或对象存储。

### 工具调用与 Agent 能力

- 增加受控工具调用，例如读取本地文件、查询数据库、生成表格、调用白名单接口。
- 为工具调用增加权限边界、执行日志和错误提示。
- 后续可抽象为 Agent 工作流，但应避免过早复杂化。

## 产品体验优化

### 聊天交互

- 增加停止生成、重新生成、复制回答、点赞/点踩等常见操作。
- 支持搜索聊天记录。
- 支持清空当前会话或批量删除会话。

### 页面体验

- 优化移动端侧边栏和聊天输入区域的布局。
- 为模型配置增加连接测试按钮。
- 对关键错误增加更醒目的提示样式，而不是只作为助手消息出现。

## 后端与部署演进

### 服务化改造

- 保留 Streamlit 作为 Demo 或管理界面，同时新增 FastAPI 后端。
- 提供 `/chat`、`/sessions`、`/models`、`/files` 等接口。
- 使用 SSE 或 WebSocket 承载流式输出。

### 部署与安全

- 增加 Dockerfile 和基础部署说明。
- 避免在页面中长期保存 API Key，后续可改为服务端配置或加密存储。
- 对上传文件、工具调用和外部接口增加大小限制和超时控制。

## 建议推进顺序

1. 增加 `.env.example`、测试和格式化工具。
2. 实现 PostgreSQL 聊天记录持久化、会话重命名和导出。
3. 增加模型服务商预设、连接测试和 API Key 加密存储。
4. 建设 Prompt 模板库。
5. 扩展 RAG 知识库问答。
6. 视使用场景决定是否引入 FastAPI 后端和多用户能力。
