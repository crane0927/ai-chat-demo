# 剩余优化与功能拓展清单

本文档用于记录当前项目后续可继续推进的优化项和功能拓展。已完成的拆分架构、渲染安全优化、未使用代码清理、模型请求高级参数、多模型配置持久化与前端切换、Prompt 模板库、PostgreSQL 聊天记录持久化、会话重命名和导出能力，以及当前版本的会话级 RAG 知识库问答不再列入本清单。

## 工程化优化

### 日志与可观测性

- 后续可扩展 token 用量统计和费用估算。

## 数据与会话能力

### 多用户支持

- 如果项目从本地 Demo 演进为团队工具，需要增加用户身份、会话归属和访问隔离。
- 可先使用简单用户标识，后续再接入正式登录体系。

## 模型与能力拓展

### 多服务商与多模型管理

- 支持同一问题调用多个模型，对比回答质量、耗时和错误情况。

### RAG 知识库演进

- 支持从“当前会话绑定文件”继续扩展到跨会话共享知识库或团队知识空间。
- 为已绑定知识文件增加移除、重建索引和批量管理能力。
- 继续优化引用展示、片段重排序、命中解释和检索效果评估。
- 视数据规模决定是否引入对象存储、专用向量库或更细粒度的索引策略。

### 工具调用与 Agent 能力

- 增加受控工具调用，例如读取本地文件、查询数据库、生成表格、调用白名单接口。
- 为工具调用增加权限边界、执行日志和错误提示。
- 后续可抽象为 Agent 工作流，但应避免过早复杂化。

## 产品体验优化

### 聊天交互

- 增加停止生成、重新生成、复制回答、点赞/点踩等常见操作。
- 支持搜索聊天记录。
- 支持清空当前会话或批量删除会话。

### 页面体验

- 优化移动端侧边栏和聊天输入区域的布局。
- 对关键错误增加更醒目的提示样式，而不是只作为助手消息出现。

## 后端与部署演进

### 服务化改造

- 保留 Streamlit 作为 Demo 或管理界面，同时新增 FastAPI 后端。
- 提供 `/chat`、`/sessions`、`/models`、`/files` 等接口。
- 使用 SSE 或 WebSocket 承载流式输出。

### 部署与安全

- 增加 Dockerfile 和基础部署说明。
- 对上传文件、工具调用和外部接口增加大小限制和超时控制。

## 建议推进顺序

1. 继续补齐知识库管理能力与检索效果优化。
2. 视使用场景决定是否引入 FastAPI 后端和多用户能力。
3. 逐步扩展工具调用与 Agent 能力，并补好权限和审计边界。
